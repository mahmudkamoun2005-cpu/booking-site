# Сайт записи/бронирования

Публичный сайт: любой посетитель может записаться **без регистрации и создания аккаунта**,
оплатить российской картой (ЮKassa) или иностранной картой (Stripe), а вы видите все заявки
в закрытой админ-панели по паролю.

## Что внутри

- `server.js` — сервер (Express)
- `db.js` — хранение заявок (SQLite, файл `data/bookings.db`)
- `public/` — сайт: `index.html` (форма записи), `admin.html` (список заявок), `success.html`
- Оплата: Stripe (карты не из РФ) и ЮKassa (карты РФ / Мир)

## 1. Запуск локально (проверить перед деплоем)

```bash
npm install
cp .env.example .env
# впишите свои значения в .env (см. раздел 3 и 4)
npm start
```

Сайт откроется на `http://localhost:3000`, админка — на `http://localhost:3000/admin.html`.

## 2. Деплой на хостинг

Проще всего — **Railway** или **Render** (бесплатный старт, не требуют сервера/VPS).

### Вариант A: Railway
1. Зарегистрируйтесь на railway.app, «New Project» → «Deploy from GitHub repo» (сначала загрузите
   эту папку в свой репозиторий на GitHub).
2. В настройках проекта → Variables добавьте все переменные из `.env.example` со своими значениями.
3. Railway сам определит Node.js проект и запустит `npm start`.
4. В настройках домена включите публичный URL (или подключите свой домен).
5. Обновите `BASE_URL` в переменных окружения на этот адрес.

### Вариант B: Render
1. render.com → New → Web Service → подключите репозиторий.
2. Build Command: `npm install`, Start Command: `npm start`.
3. Добавьте переменные окружения (Environment) из `.env.example`.
4. Render выдаст `https://ваш-сайт.onrender.com` — впишите его в `BASE_URL`.

### Вариант C: свой VPS
1. Установите Node.js 18+, склонируйте проект, `npm install --production`.
2. Запускайте через `pm2` (`pm2 start server.js --name booking`) для автоперезапуска.
3. Настройте Nginx как обратный прокси на порт 3000 и получите SSL-сертификат (Let's Encrypt/certbot) —
   без HTTPS платёжные системы работать не будут.

**Важно:** база SQLite — это файл на диске. На Railway/Render на бесплатном тарифе диск может
пересоздаваться при передеплое — для продакшена подключите постоянный volume (обе платформы это
поддерживают) или замените SQLite на облачную БД (Postgres), если заявок будет много.

## 3. Оплата иностранными картами — Stripe

1. Зарегистрируйтесь на stripe.com (можно как физлицо/ИП, привязка к юрлицу нужна для вывода денег
   на счёт — уточняйте требования по вашей стране при регистрации).
2. В Dashboard → Developers → API keys скопируйте **Secret key** → `STRIPE_SECRET_KEY`.
3. Developers → Webhooks → Add endpoint:
   - URL: `https://ваш-домен/webhook/stripe`
   - Событие: `checkout.session.completed`
   - Скопируйте **Signing secret** → `STRIPE_WEBHOOK_SECRET`.
4. Пока не подключили реальный приём денег — используйте тестовые ключи (`sk_test_...`) и тестовую
   карту `4242 4242 4242 4242`, любой будущей датой и любым CVC.

## 4. Оплата российскими картами — ЮKassa

1. На yookassa.ru нужна регистрация **ИП или ООО** (для физлиц без статуса ИП приём платежей по
   закону РФ не оформить — это требование самой платёжной системы, не наше). Есть похожие сервисы
   (CloudPayments, Тинькофф Касса) с аналогичным условием.
2. После регистрации: Личный кабинет → Настройки → API-ключи:
   - `shopId` → `YOOKASSA_SHOP_ID`
   - `Секретный ключ` → `YOOKASSA_SECRET_KEY`
3. Там же включите уведомления (webhook) на `https://ваш-домен/webhook/yookassa`, событие
   `payment.succeeded`.
4. Для тестов ЮKassa выдаёт тестовый магазин с тестовыми ключами — тестовая карта указана прямо
   в их документации в личном кабинете.

## 5. Email/СМС-уведомления клиенту

После подтверждения оплаты клиент автоматически получает email и/или СМС с деталями записи.
Всё опционально — что не настроите, просто не будет отправляться (ошибок не будет).

**Email** — любой SMTP:
- Яндекс: `smtp.yandex.ru`, порт `465`, логин — почта, пароль — «пароль приложения»
  (создаётся в настройках Яндекс ID → Пароли и авторизация, обычный пароль от почты не подойдёт).
- Gmail: `smtp.gmail.com`, порт `465`, тоже нужен «пароль приложения» (App Password).
- Или сервис для рассылок: SendGrid, Mailgun — у них есть готовые SMTP-настройки.

**СМС на российские номера** — через [SMS.ru](https://sms.ru/?panel=api):
1. Зарегистрируйтесь, пополните баланс.
2. Скопируйте `api_id` из личного кабинета → `SMS_RU_API_ID`.

**СМС на иностранные номера** — через [Twilio](https://console.twilio.com):
1. Зарегистрируйтесь, купите номер для отправки СМС.
2. `Account SID` и `Auth Token` с главной страницы консоли → `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`.
3. Купленный номер (в формате `+1...`) → `TWILIO_FROM_NUMBER`.

Логика простая: заявки, оплаченные в рублях, получают СМС через SMS.ru; оплаченные в долларах —
через Twilio. Email уходит через SMTP независимо от валюты, если клиент указал адрес.

## 6. Админ-панель

- Откройте `/admin.html`, введите пароль из `ADMIN_PASSWORD`.
- Список заявок обновляется по кнопке «Обновить».
- Пароль — простая защита для одного администратора. Если админов несколько или нужны разные права
  доступа — скажите, добавлю полноценные логины.

## 7. Что стоит донастроить перед реальным запуском

- **Часовой пояс и рабочие часы** — сейчас слоты 9:00–18:00 по часу, задаются в `db.js` (`WORK_HOURS`).
- **Список услуг и цены** — тоже в `db.js`, массив `SERVICES`.
- **HTTPS обязателен** — Stripe и ЮKassa не примут вебхуки по http (Render/Railway дают HTTPS сразу).
- **Неоплаченные заявки** — если клиент дошёл до оплаты, но не завершил её, слот остаётся занятым
  («висит» в статусе `pending`). Для старта это не критично, но при большом потоке заявок стоит
  добавить автоматическое снятие брони через N минут — скажите, если нужно.

## 8. Отмена записи и возврат оплаты

В админке у каждой заявки есть кнопка «Отменить»:
- если заявка была оплачена — деньги возвращаются автоматически (Stripe refund или возврат ЮKassa)
- слот сразу освобождается и снова становится доступен для записи на сайте
